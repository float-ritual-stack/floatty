# Agent Types

## Outline-native agents

Operate directly on the outline as shared cognitive substrate.

Examples:
- Clerk
- Librarian
- Gardener
- Renderer

Typical work:
- ingestion
- retrieval
- refinement
- projection
- memory organization
- attribution / work logging

## External execution agents

Operate primarily in shell / repo / external execution contexts.

Examples:
- Claude Code in xterm
- implementation agents
- debugging / testing agents
- git / shell-driving agents

Typical work:
- edit code
- run tests
- inspect logs
- manage branches / commits
- build applications, including floatty itself

## Relationship

External execution agents perform work.
Outline-native agents organize, connect, and project work.

Execution agents may write traces, plans, issue links, summaries, and artifacts back into the outline.

## Rule

Floatty complements execution surfaces.
It does not replace them.
